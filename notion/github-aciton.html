原理：Rclone/Gclone 的sync功能 实现Google Drive / onedrive 的备份
1、注册GitHub，并Fork本项目https://github.com/artxia/Aciton-Rclone
2、获得service account文件，并打包成accounts.zip上传到能下载的地方，然后记录下载url
（可以直接上传到自己的GoogleDrive，点分享，然后打开分享链接点下载能获得一个这样的链接：https://drive.google.com/uc?id=1xBebirSOV99XXXXXXXXXXXXXXA5n5ZP&export=download）
3、选择好需要备份的云盘目录folde_id1和备份目的地目录folde_id2，并记录下目录id
4、打开刚才Fork的项目，找到Settings（设置）然后点Secrets（隐私）新建如下内容
GCLONE_CONFIG
[gc] type = drive scope = drive service_account_file = /home/runner/.config/rclone/sa/03327xxxxxxxxxxxxx55523.json service_account_file_path = /home/runner/.config/rclone/sa/ root_folder_id = root
Shell
GCLONE_SA_DLURL
https://drive.google.com/uc?id=1xBebirSOV99XXXXXXXXXXXXXXA5n5ZP&export=download
Shell
GCLONE_PATCH
gc:{folde_id1} gc:{folde_id2}/media/ --drive-server-side-across-configs --transfers 16 --checkers 32 --checksum --stats-one-line-date -q
Shell
5、在你自己的GitHub项目里面，点Actions然后点左侧gclone sync切换，然后点 Run workflow 开始第一次同步。(看不到Auto Install的话，点开yml文件随便加一空行保存)
（首次同步耗时较长，但GitHub最多运行时间是6小时，所以项目需要自启好几次耗时几天才能完成初次同步）。项目默认是每六小时自动运行一次同步任务，第一次完成后可自行修改同步时间。
6、多盘之间相互同步实现分流。首先复制gclone_sync.yml文件，并修改命名为gclone_sync1.2.3....然后建议采用1→2→3→4这样的架构来同步这四个盘，申请几千个sa全部放一个group邮箱下，每 100个sa建一个文件夹打包成accounts1.zip、accounts2.....分别给gclone sync 1、gclone sync 2...使用。这样就能实现多盘多线路运行备份。
